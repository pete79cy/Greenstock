The **red toast** is the browser itself complaining – it never even tried to talk to your server:

```
Error
Window.fetch: Invalid request method /api/payslips
```

That message appears only when **the 2nd argument you pass to `fetch()` is a plain string**.
The runtime thinks you’re trying to use the string as the HTTP verb, realises that’s illegal, and aborts the call.
Result: nothing is sent, nothing is deleted/created, no Network-tab entry is recorded.

---

## 1 — Find every mis-wired call

```bash
# from project root
grep -R "fetch(" client | grep -E "'(GET|POST|PUT|PATCH|DELETE|HEAD)'" 
```

You’ll see lines like:

```ts
// ❌ wrong
fetch("/api/employees/" + id, "DELETE")

// ❌ wrong
fetch("/api/payslips", "POST")
```

(or the same with double quotes).

---

## 2 — Patch them

```ts
// ✅ correct pattern
await fetch(`/api/employees/${id}`, {
  method: "DELETE",
  credentials: "include",
  headers: { Accept: "application/json" },
});

// ✅ payslip creation
await fetch("/api/payslips", {
  method: "POST",
  credentials: "include",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    employeeId,
    payPeriod,
    payDate,
    grossSalary,
    notes,
  }),
});
```

### Tip – central helper to avoid this again

```ts
// client/src/lib/api.ts
export const api = (
  url: string,
  options: RequestInit & { method?: string } = {},
) =>
  fetch(url, {
    credentials: "include",
    headers: { Accept: "application/json", ...options.headers },
    ...options,
  });
```

Then everywhere:

```ts
await api(`/api/employees/${id}`, { method: "DELETE" });
await api("/api/payslips", { method: "POST", body: JSON.stringify(data) });
```
