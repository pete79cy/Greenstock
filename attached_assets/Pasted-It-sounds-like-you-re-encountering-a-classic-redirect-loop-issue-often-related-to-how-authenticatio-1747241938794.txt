It sounds like you're encountering a classic redirect loop issue, often related to how authentication state is managed and checked in your application. This typically happens when:

1.  You log in successfully.
2.  The app tries to redirect you to a protected page (your main page).
3.  The protected page checks if you're authenticated.
4.  For some reason (e.g., the authentication state hasn't updated quickly enough, or it's not being checked correctly), it thinks you're *not* authenticated.
5.  So, it redirects you back to the login page.

To fix this, we need a robust way to manage and check the authentication status across your app. I'll implement a common solution using React Context for authentication.

Here's the plan:

1.  **Create an `AuthContext`**: This will hold the authentication state (e.g., if the user is logged in and their token/details) and provide functions to log in and log out.
2.  **Create a `ProtectedRoute` component**: This component will wrap routes that require authentication. If the user isn't logged in, it will redirect them to the login page.
3.  **Create a `LoginPage` component**: A simple page with a login form. For now, it will simulate a login.
4.  **Update `App.tsx`**: We'll wrap our routes with an `AuthProvider` (from `AuthContext`) and use the `ProtectedRoute` for the main page (`/`).
5.  **Modify `Index.tsx` (your main page)**: Ensure it can potentially use authentication details if needed, and add a logout button.

import React, { createContext, useContext, useState, useEffect, ReactNode } from 'react';
import { useNavigate } from 'react-router-dom';

interface AuthContextType {
isAuthenticated: boolean;
isLoading: boolean;
userToken: string | null;
login: (token: string) =\> void;
logout: () =\> void;
}

const AuthContext = createContext\<AuthContextType | undefined\>(undefined);

export const AuthProvider: React.FC\<{ children: ReactNode }\> = ({ children }) =\> {
const [isAuthenticated, setIsAuthenticated] = useState\<boolean\>(false);
const [userToken, setUserToken] = useState\<string | null\>(null);
const [isLoading, setIsLoading] = useState\<boolean\>(true); // To handle initial auth check
const navigate = useNavigate();

useEffect(() =\> {
console.log('AuthProvider: Checking initial authentication status.');
const storedToken = localStorage.getItem('userToken');
if (storedToken) {
console.log('AuthProvider: Token found in localStorage.');
setUserToken(storedToken);
setIsAuthenticated(true);
} else {
console.log('AuthProvider: No token found in localStorage.');
}
setIsLoading(false);
}, []);

const login = (token: string) =\> {
console.log('AuthContext: login function called.');
localStorage.setItem('userToken', token);
setUserToken(token);
setIsAuthenticated(true);
console.log('AuthContext: User authenticated, navigating to /');
navigate('/'); // Navigate to main page after login
};

const logout = () =\> {
console.log('AuthContext: logout function called.');
localStorage.removeItem('userToken');
setUserToken(null);
setIsAuthenticated(false);
console.log('AuthContext: User logged out, navigating to /login');
navigate('/login'); // Navigate to login page after logout
};

return (
\<AuthContext.Provider value={{ isAuthenticated, userToken, login, logout, isLoading }}\>
{children}
\</AuthContext.Provider\>
);
};

export const useAuth = (): AuthContextType =\> {
const context = useContext(AuthContext);
if (context === undefined) {
throw new Error('useAuth must be used within an AuthProvider');
}
return context;
};
