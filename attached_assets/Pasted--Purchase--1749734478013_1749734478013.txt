δύο νέα μοντέλα στη βάση δεδομένων σας:

    Purchase (Αγορά/Παραγγελία): Αυτό το μοντέλο θα καταγράφει κάθε ξεχωριστή παραγγελία.
        purchaseId: Μοναδικό αναγνωριστικό.
        supplier: Όνομα προμηθευτή.
        purchaseDate: Ημερομηνία παραγγελίας.
        arrivalDate: Ημερομηνία παραλαβής.
        totalCost: Συνολικό κόστος αγοράς των φυτών.
        shippingCost: Κόστος μεταφορικών.
        customsCost: Κόστος εκτελωνισμού/φόρων.
        currency: Νόμισμα (π.χ., EUR, USD).
        notes: Σημειώσεις για την παραγγελία.

    PurchasedPlant (Αγορασμένο Φυτό): Αυτό θα συνδέει κάθε φυτό με μια συγκεκριμένη αγορά.
        purchasedPlantId: Μοναδικό αναγνωριστικό.
        purchaseId: Σύνδεση με την παραπάνω παραγγελία.
        plantName: Όνομα φυτού (ή plantId αν έχετε ήδη κατάλογο φυτών).
        quantity: Ποσότητα.
        costPerUnit: Κόστος ανά μονάδα από τον προμηθευτή.
        status: Κατάσταση κατά την παραλαβή (π.χ., "Άριστη", "Καλή", "Μέτρια").

Νέα API Endpoints

Θα δημιουργήσετε ένα νέο αρχείο routing, π.χ., server/routes/purchases.ts, με τις εξής διαδρομές:

    POST /api/purchases: Για τη δημιουργία μιας νέας καταχώρησης αγοράς.
    GET /api/purchases: Για την ανάκτηση της λίστας όλων των αγορών (για την ιστορικότητα και τον προγραμματισμό).
    GET /api/purchases/:id: Για την ανάκτηση των λεπτομερειών μιας συγκεκριμένης αγοράς, μαζί με όλα τα φυτά που περιλαμβάνει.
    PUT /api/purchases/:id: Για την ενημέρωση μιας υπάρχουσας αγοράς.

## 2. Επεκτάσεις στο Frontend (Client Side)

Στο React frontend, θα δημιουργήσετε νέα components για τη διαχείριση αυτών των δεδομένων.
Νέα UI Components

    PurchaseForm (/components/forms/PurchaseForm.tsx): Μια φόρμα για να καταχωρείτε μια νέα αγορά. Θα περιλαμβάνει πεδία για τον προμηθευτή, τις ημερομηνίες, τα κόστη, και μια δυναμική λίστα όπου θα μπορείτε να προσθέτετε τα φυτά της παραγγελίας (όνομα, ποσότητα, κόστος).

    PurchasesList (/pages/PurchasesPage.tsx): Μια νέα σελίδα που θα εμφανίζει έναν πίνακα με όλες τις προηγούμενες αγορές. Αυτό θα λειτουργεί ως το εργαλείο προγραμματισμού, επιτρέποντάς σας να βλέπετε το ιστορικό παραγγελιών, τις τιμές ανά προμηθευτή και τις ποσότητες.

    PurchaseDetails (/pages/PurchaseDetailsPage.tsx): Μια σελίδα που θα εμφανίζει όλες τις λεπτομέρειες μιας αγοράς. Εδώ θα υλοποιηθεί το εργαλείο κοστολόγησης και ανάλυσης:
        Ανάλυση Κόστους: Θα εμφανίζει το συνολικό κόστος της παραγγελίας (αγορά + μεταφορικά + τελωνείο). Το πιο σημαντικό, θα υπολογίζει αυτόματα το τελικό κόστος ανά φυτό, κατανέμοντας τα επιπλέον έξοδα σε κάθε μονάδα.
        Ανάλυση Πορείας Φυτών: Σε αυτή την προβολή, θα μπορούσατε να προσθέσετε τη δυνατότητα να ενημερώνετε την κατάσταση των φυτών με την πάροδο του χρόνου (π.χ., ποσοστό επιβίωσης, προβλήματα προσαρμογής), συνδέοντας την αρχική τους κατάσταση με την εξέλιξή τους.

## Πώς Υλοποιεί τους Στόχους σας

    Καταγραφή Αγορών: Γίνεται μέσω της PurchaseForm.
    Εργαλείο Προγραμματισμού: Η PurchasesList σας δίνει το ιστορικό δεδομένων που χρειάζεστε για να προγραμματίσετε μελλοντικές παραγγελίες.
    Εργαλείο Κοστολόγησης: Η PurchaseDetails κάνει αυτόματα τον υπολογισμό του τελικού κόστους ανά φυτό.
    Ανάλυση Πορείας: Παρακολουθείτε την εξέλιξη των φυτών από την παραλαβή τους και μετά, επιτρέποντάς σας να αξιολογείτε προμηθευτές και την ποιότητα των φυτών τους.
